# 悄悄话 V2 产品计划

> 决策日期：2026-07-13
> 状态：可运行垂直基线已实现，v11–v22 与客户端均为未发布候选；完整 MVP 增强与真机验收继续推进
> 适用范围：iPhone、iPad、Fastify 服务端、PostgreSQL、AI Agent

## 1. 已确认的产品边界

- 当前 App 只侧载到两位现有用户的设备，但数据模型和接口不得再写死 `xu / si`。
- 未来允许其他情侣注册、用邀请码配对；每个账号当前只加入一个有效情侣空间。
- 同一账号可同时登录 iPhone 和 iPad，消息、设置、壁纸、贴纸、收藏与所有业务数据都要同步。
- 已读回执和在线状态不可关闭。已读严格表示：聊天页处于可见状态，而且消息已经展示。
- 撤回执行完整删除：消息正文、附件、缩略图、转写、搜索索引、引用预览、Memory 证据和相册引用一起清理。
- 通知继续使用 Bark。共享提醒通知双方的所有有效设备；私人提醒只通知创建者的所有有效设备。
- 备份保持轻量：数据库与媒体快照、轮换保留、可验证恢复，不建设复杂灾备平台。
- 开发优先级是功能完整度；身份、所有权、同步、媒体和通知底座仍是避免功能返工的必要前置。

## 2. 产品定位

悄悄话不只是聊天工具，而是两个人共同生活的长期容器：聊天产生素材，计划推动行动，照片留下结果，Memory 帮助大橘理解关系，大橘再把这些内容组织成轻量的共同体验。

核心闭环：

```text
聊天 ──语音/照片/约定──> 转写、相册、计划
  │                           │
  └──────────────> Memory <───┘
                         │
                  大橘“今天一起”
                         │
                  藏品 / 时光 Moment
```

当前候选已经落地身份/设备/所有权、Sync V2 补拉、Memory 控制中心、语音转写、聊天媒体入册相册、那年今日、共享/私人日历和共同宠物。下文的直接媒体上传、离线 mutation queue、长语音按句定位、月末窝记、跨域联动解锁、iPad 拖放/双栏和完整无障碍矩阵仍是产品目标，不应理解为已完成或已发布。

## 3. 一级信息架构

| 一级入口 | 内容 |
|---|---|
| 聊天 | 情侣聊天、回复/搜索、语音与转写、聊天媒体 |
| 时光 | 那年今日、共同相册、纪念日、大橘日记、聊天回顾 |
| 大橘 | 共同养成小窝、“今天一起”、藏品、AI 私聊入口 |
| 计划 | 日程、共享/私人提醒、备忘、约会提议 |
| 我的 | 账号与设备、主题、Bark、存储、收藏、Memory 控制中心 |

“记录”改名为“时光”，“提醒”扩展为“计划”；仍保持五个一级入口，不增加第六个 Tab。

## 4. 视觉与交互方向

主题是“漫长悄悄话留下的共同痕迹”，而不是通用粉色卡片模板。

- 背景保留温柔浅色，但减少整屏渐变和每块内容都套玻璃卡的做法。
- 玻璃只用于 Tab、工具栏、悬浮控制；照片、日历、表单和长列表用更稳定的实色表面。
- 品牌签名为“成双回声”：蓝、粉两条细线在真正共同完成的节点汇合成橘色爪印。
- 建议语义色：夜谈墨 `#302A2A`、雾桃纸 `#FFF8F7`、橘光 `#F28C52`、小旭蓝 `#5B7FE8`、小偲粉 `#EF6688`、共鸣紫 `#8F72D8`。
- 正文继续使用系统字体；日期、时长采用等宽数字。只有“那年今日”、月份和重要纪念数字使用克制的展示字体。
- 大橘采用有轮廓辨识度的 2D 角色与窗边小窝，不继续使用 emoji 占位，也不急于引入质量不稳定的 3D 模型。

## 5. Memory 控制中心

入口：`我的 → 大橘与记忆`，副标题“查看、纠正或忘掉记忆”。

### 用户能力

- 在“全部 / 共同 / 我的”之间切换。
- 按事实、经历、计划、近况、关系、理解分类并搜索自然语言内容。
- 查看记忆正文、范围、最近更新时间和来源对话。
- 手动纠正内容、标为重要、让大橘彻底忘掉。
- 分别触发“整理共同聊天”和“整理我的私聊”。
- 后续从来源证据跳回原消息，并支持长按消息“这条不让大橘记住”。

界面不暴露 embedding、置信度、内部 key 等工程字段。

### MVP 验收

- 共同记忆双方可管理，私聊记忆只属于本人。
- 新增、编辑、删除在手机和平板同步。
- 删除后不会被同一证据再次生成；撤回来源消息会同步删除相关证据与失去来源的记忆。
- 搜索、分类、来源、缓存、错误保留与重试均可用。

### 当前实现进度

- 已加入正式登录态 API：列表/搜索、范围隔离、cursor 分页、来源、版本冲突、纠正、删除 exclusion 和立即整理。
- 已加入设置入口、iOS 控制中心和 Sync V2 跨设备刷新。
- 待补来源消息跳转、本地离线缓存，以及新情侣完整 Agent/MCP 自动 Memory。

## 6. 共同相册与“那年今日”

入口：`时光 → 回忆 / 相册`。

### 回忆首页

- 顶部展示“那年今日”；多年份内容横向浏览。
- 最近照片用一大两小的内容优先拼图。
- 保留纪念日、大橘日记和聊天回顾，但统一到同一时间线。
- 没有往年内容时提供“回看这个月”，不伪造空回忆。

### 相册

- 全部时光、聊天照片、收藏、自建相册。
- 图片、视频、Live Photo；按年月分组、游标分页、自适应网格。
- 双方可各写一条注脚；支持封面、移动、多选、iPad 拖放。
- 聊天媒体菜单增加“加入共同相册”，以资产 ID 去重。
- 上传显示逐项进度、失败重试和取消；离线进入可靠队列。

### MVP 验收

- 三类媒体可上传、查看、分页和完整删除。
- 聊天媒体无重复加入相册；双方增删、封面和注脚实时同步。
- “那年今日”按用户时区稳定生成，2 月 29 日在非闰年于 2 月 28 日展示。
- 一万项媒体仍使用缩略图、懒加载和游标分页。

## 7. 共享日历与计划

入口：`计划 → 日程 / 提醒 / 备忘`。

默认页是紧凑周条加按天分组的 Agenda；工具栏可切完整月历。筛选为“全部 / 我们 / 我的”。

- `+` 菜单创建新日程、提醒或备忘。
- 共享日程默认双方参与；私人内容只属于创建者。
- 共享提醒通过 Bark 通知双方所有有效设备，私人提醒只通知创建者设备。
- 不做周期提醒、稍后提醒和安静时段。
- 聊天消息和大橘确认卡可以创建计划，并保留来源消息链接。
- “约会提议”允许给出 2–3 个时间供对方选择；普通共享日程不需要审批。
- 完成计划可以进入时光、关联相册并解锁大橘藏品。

### MVP 验收

- 月历、周条和 Agenda 使用同一份数据，全天、跨天、时区与夏令时正确。
- 离线创建、编辑、完成、取消和删除均不会丢失。
- 并发修改返回冲突并展示最新版本，不静默覆盖。
- iPad 宽屏同时展示日期导航和日程列表。

## 8. 语音转写

MVP 采用“语音发送成功后，服务端异步转写”，转写失败不得影响原语音发送。

- 气泡状态：转成文字、正在转写、完成、未听清、失败可重试。
- 结果默认折叠一至三行；长语音按句分段，点击句子跳到对应播放时间。
- 用户可以纠正文字；搜索、引用和 Memory 使用修正版，并标记来源为语音转写。
- 设置提供“自动转写收到的语音”和“自动转写我发送的语音”；默认前者开、后者关。
- 离线时排队；撤回时音频、转写和索引一起删除。

### MVP 验收

- 成功、无清晰语音、超时、重试和撤回路径完整。
- 修正版跨设备同步并进入搜索。
- VoiceOver 能读出时长、状态、转写和操作。
- 展开长文字不破坏聊天滚动锚点。

## 9. 大橘共同养成系统

大橘的定位是“共同生活的见证者”，不是需要打卡照顾的压力型电子宠物。删除饱食、清洁、心情和精力条；长期不打开时，大橘只会睡觉或晒太阳，不生病、不掉亲密度。

### 核心循环

1. 大橘每天提供一个轻量“今天一起”，例如各自拍一张天空、各选一首歌或回答一个小问题。
2. 两人可异步回应；蓝粉两条回声完成后汇合成橘色爪印。
3. 服务端只结算一次与内容有关的“小窝藏品”，不发通用金币。
4. 藏品可以布置在窗边小窝，同时生成时光 Moment。
5. 每月底生成“月末窝记”，由双方确认后进入共同相册。

摸头、碰爪和逗猫棒等互动不设置“次数不足”；重复操作可以减少奖励，但不能禁止互动。计划完成、相册 Moment 和双人回答都能解锁藏品。

### 界面

- iPhone：上半屏为窗边小窝与 2D 大橘，下半屏为“今天一起 / 最近藏品 / 最近足迹”内容抽屉。
- iPad：左侧约 60% 为场景，右侧 360–420pt 内容栏；布置模式支持拖放和指针。
- 所有宠物手势都有等价按钮；Reduce Motion 下关闭持续漂浮、弹跳和大范围位移。

### MVP 验收

- 名字、场景、藏品、动作和当前题目跨重启、跨设备保持。
- 两人回应后只结算一次，离线补交不会重复奖励。
- 长期未使用没有惩罚性状态或催促通知。
- 场景素材加载失败时回退到静态插画，核心操作仍可使用。

## 10. iPhone 与 iPad

### iPhone

- 五个系统级 Tab，各自维护 NavigationStack。
- 月历、相册详情和 Memory 详情使用 push；编辑器按内容复杂度使用 sheet。
- 常用操作位于易触区域或标准工具栏，点击区域至少 44×44pt。

### iPad

- 根导航使用可转换 Tab/侧栏，不放大当前自绘手机底栏。
- 聊天：会话列表 + 对话详情。
- 时光：回忆/相册导航 + 网格或媒体详情。
- 计划：迷你月历与筛选 + Agenda；超宽时显示检查器。
- Memory：分类列表 + 记忆详情。
- 大橘：小窝场景 + 固定内容栏。
- 窗口变窄时按可用宽度折叠，不能只按设备型号判断。
- 支持横屏、Split View、Stage Manager、指针、照片拖放和基础键盘快捷键。

## 11. 成熟度与无障碍门槛

- Dynamic Type 覆盖至 AX5；大字号下网格转列表、横排操作转纵排。
- VoiceOver 有正确标题层级、焦点顺序、图片描述、语音文字和宠物替代操作。
- 状态不能只依赖蓝/粉颜色，必须同时有头像、名字、图标或文字。
- 深色、提高对比度、减少透明度、减少动态效果分别测试。
- 加载优先展示缓存或骨架；错误保留已有内容并明确提供下一步。
- 所有系统状态必须真实，不用“已送达”“已读”等乐观文案掩盖未确认操作。

## 12. 剩余开发顺序

1. macOS CI 修完 Swift 编译/结构问题，完成 v10 → v22 隔离迁移、备份恢复和回滚演练。
2. 将完整 Agent/MCP/Memory runtime 全量迁移到 conversation/account/couple ownership。
3. 增加 `sync:available` 唤醒、preference 云同步、cursor 过期重建和通用 mutation 幂等。
4. 相册补直接选择/拍摄上传、多选/移动；转写补自动策略、长语音分句与播放定位。
5. 日历补离线 mutation queue、周条/Agenda；宠物补月末窝记、计划/相册联动和 iPad 拖放。
6. 执行双设备、iPad、Dynamic Type、VoiceOver、深色/高对比和失败恢复整体验收，再开放其他情侣。

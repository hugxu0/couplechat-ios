# ADR-0001：客户端与服务端保持单仓库

- 状态：Accepted
- 日期：2026-07-16

## 背景

本地曾被整理成两个无 Git 历史的目录，文档又声称存在两个独立远程仓库；实际 GitHub 权威历史仍是包含 `Sources/` 与 `server/` 的单仓库，独立 server 仓库不存在。两端共享 REST、Socket、Sync 和数据库兼容关系，拆仓会增加契约漂移和 AI 交接成本。

## 决策

保持单仓库。客户端与服务端可以独立测试、构建和发布，但协议变化必须在同一提交完成。

服务端发布只打包 `server/` 子树，使用独立 `server-vX.Y.Z` tag 或完整 commit SHA、metadata 和 SHA-256；生产主机不需要复制 iOS 源码。

## 影响

- 优点：原子修改两端、统一 CI、保留历史、AI 只需一个事实源。
- 代价：workflow 必须用 path filter 和明确工作目录，版本/发布状态必须分别记录。
- 禁止：再次通过复制目录“模拟拆仓”、把两个无 Git 快照当成独立权威仓库。

# 当前已知问题

这里只保留仍会影响当前使用、数据安全或生产安全的问题。已经修复但只等待旧 CI 证明的条目、未计划的未来功能和维护性愿望不再列入清单。

## 当前使用风险

### SYNC-002：同毫秒消息的历史分页不是稳定游标

同一毫秒内有多条消息时，只有 `ts` 的 `after/before/since` 可能重复、停滞或跳过。当前聊天仍依赖现有分页和 Sync V2；修复需要统一使用 `(ts,id)` 复合游标或服务端序号。

### IOS-003：开发构建默认连接生产

Debug 与 Release 当前都使用 `https://hoo66.top`。在没有独立开发服务前，开发构建不能执行破坏性测试或批量写入。

### IOS-004：部分大文件仍可能在主线程读取

附件、壁纸和部分媒体缓存路径仍使用 `Data(contentsOf:)`，最大文件可达 50 MB，极端情况下可能造成界面冻结或内存峰值。

### IOS-007：快速切换账号仍有旧异步任务竞态

登出、SQLite close、下一账号 open 和 Socket 任务没有统一 session generation。快速切换账号时，旧任务晚到可能影响新账号内存或数据库。

### UPLOAD-001：崩溃遗留临时文件可能阻塞备份

`.uploading` 临时文件仍需要可靠的超时和活跃 lease 判断。备份遇到不安全的遗留文件时会停止，不能直接忽略。

## 生产安全风险

### SEC-001：认证入口的滥用防护仍有限

可信代理边界、边缘与应用限流、退避、异步密码校验和压力验证仍需在私有运维环境确认。具体阈值不写入公开仓库。

### SEC-002：密钥用途与轮换仍较简单

会话、内部调用和媒体用途尚未完全拆分，密钥轮换前仍需确认设备会话和历史媒体兼容性。

### SEC-003：媒体授权与日志脱敏仍需加强

媒体访问和两层 Nginx/应用日志仍需继续核对短期授权、查询参数脱敏和保留期。

### OPS-002：离机备份状态只能在私有环境确认

公开仓库不能证明供应商、容量、最近成功批次或恢复演练。生产发布前仍需由私有运维资料确认当前可用的加密离机副本。

### OPS-003：应用回滚不等于数据库回滚

迁移只前进，旧镜像不一定兼容新 schema。schema 变化后不能只切换旧镜像，必须使用兼容镜像或同一批次的数据库、uploads 和必要密钥恢复。
